# 保証層の正直さ: 診断が主張できること・できないこと

このドキュメントは**分析資料(構想)**です。プリフライト診断([DESIGN.md](DESIGN.md) §9)が「枠内かを保証する」と言うとき、その保証がどこまで正直かを点検します。実装を拘束するのは DESIGN.md と ROADMAP のみ。ここで特定された「報告書に刻むべき項目」は、採用時に DESIGN.md と Fit Report 仕様([ECOSYSTEM_VISION.md](ECOSYSTEM_VISION.md) §6 → 将来の FIT_REPORT_FORMAT)へ昇格させます。関連: [FAMILY_MODEL.md](FAMILY_MODEL.md)。最終更新: 2026-06-13。

## 0. 一行で

保証層の最大のリスクは**偽りの確信(false confidence)** — 緑/黄/赤という小綺麗な判定が、(1)場所で変わるしきい値、(2)計器の誤差、(3)静的な視野、を隠すこと。No Cache(監査可能性)を価値の柱に置く以上、判定は自分の誤差と限界まで申告して初めて信用される。これは「赤なら他ツールへ送り出す正直なルーター」の信頼性の土台でもある。

## 1. 包絡は定数でなく「場」

§9 を1つの大域しきい値(〜1cm、半径の 10〜15%)として扱ってきたが、ファミリー構造([FAMILY_MODEL.md](FAMILY_MODEL.md))を認めると怪しい。修正可能包絡は採寸空間の上の「**場**」で、場所ごとに違う可能性が高い:

- 小柄系は局所半径が小さく曲率も急 → 許容はもっと小さいかも
- 大柄系は半径が大きく、同じ深さでも浅い扱い

半径% で正規化しても、曲率・布の張り・部位で許容は動く。**しきい値はファミリーごと/部位ごとであるべきかもしれない**。そして実機 E2E はある1つのファミリー1点の較正にすぎない — 大域への一般化は仮説。較正の意味が「v1 の物差し作り」から「**場の1点を採って外挿可能性を見る**」へ変わる。

## 2. 計器に誤差がある → 出すべきは判定でなく信頼区間

体はカプセル近似(Humanoid ボーン由来)。だが太ももはカプセルでないし、胴は全くカプセルでない(肩甲骨・鎖骨・非凸)。**貫通深度には、しきい値較正とは独立の「モデル誤差」が乗る**。フェーズ3の半径自動推定でも消えない(凸近似の限界)。

帰結: 境界付近では誤差バーが緑/黄/赤の線をまたぐ。**正直な診断は「黄」でなく「黄(赤の可能性)」を出すべき** — 単一判定値より信頼区間。誤差を併記する科学の測定と同じで、誤差を隠した小綺麗な単一判定こそ false confidence の温床。Fit Report に信頼度欄を持たせる([ECOSYSTEM_VISION.md](ECOSYSTEM_VISION.md) §6 拡張)。

## 3. しきい値もアバターも漂流する → 標準は腐る

二重の漂流:

- **しきい値**: 較正が進むと §9 は動く。v1 で「緑」だった報告書が v2 で「黄」に化ける → 判定は**しきい値バージョンと必ず対**(§6)。帰結は「過去の判定は後から覆る」
- **アバター**: 作者が代表的なアバターを更新する(メッシュ変更)。ハッシュが変われば旧報告書は陳腐化 — ファミリーの重心([FAMILY_MODEL.md](FAMILY_MODEL.md) §7)それ自体が時間で動く

帰結: Fit Report は「読める」だけでは不足、**再実行可能(re-runnable)**でなければならない — メッシュハッシュ・計測条件・しきい値版を全部刻み、誰でも今の版で測り直せる。鮮度は中央権威でなく**再現性で**担保する(偽装が自滅的なのと同じ仕組み; §6)。標準は作って終わりでなく腐敗との戦い。

## 4. 診断は静的、痛みは動的

診断は静的ポーズ(か数個)で測る。だがユーザーが最も気にする貫通は、肘・膝・肩などよく動く継ぎ目に集中する([DEFORMATION_METHODS.md](DEFORMATION_METHODS.md) の「肌見せ境界 × 関節」):

- 立ち姿で緑でも、座る/踊る/腕を上げるで赤になりうる。**静的緑は必要条件であって十分条件でない**
- physbone/布物理(スカート・髪)は実行時に動く。静的メッシュの診断はそこに何も言えない

帰結: 「ポーズ空間をどう・何点サンプリングするか」は未解決の設計軸で、**保証の正直さがそこに懸かる**。フェーズ3の代表ポーズがその入口。「保証」を名乗る以上、**どのポーズ範囲での保証かを報告書に明記**する。動的を当面スコープ外と境界を引くのも誠実な一手。

## 5. 統一と DESIGN への含意

1・2・4 は「静的な保証がどれだけ正直か」、3 は「保証が時間でどう腐るか」 — 全て **false confidence リスク**の側面。皮肉なことに、正直なルーター(赤で他ツールへ送り出す挙動)が信用されるのは、診断が自分の誤差と限界まで申告するからこそ。

報告書(§6 → FIT_REPORT_FORMAT)へ昇格すべき項目:

- しきい値バージョン / ツールバージョン
- メッシュハッシュ(アバター・衣装)
- 計測条件(margin・ポーズ・シェイプキー)
- 判定の信頼度 / 誤差(§2)
- 保証ポーズ範囲(§4)
- 再実行に必要な情報一式(§3)

# アバター・ファミリーと採寸空間

このドキュメントは**分析資料(構想)**です。「どの体型差までなら着回しが貫通修正だけで成立し、どこからリターゲティングになるか」を、採寸空間という1つの枠組みで説明します。実装を拘束するのは [DESIGN.md](DESIGN.md) と ROADMAP のみで、本書の概念は採用が決まった部分を DESIGN.md(主に §9 サポート範囲)へ昇格させます。関連: [ECOSYSTEM_VISION.md](ECOSYSTEM_VISION.md)(§4 サイズタイリング・§5 採寸表)、[DIAGNOSTIC_HONESTY.md](DIAGNOSTIC_HONESTY.md)(本書のクラスタ境界が時間と計測でどう揺らぐか)。最終更新: 2026-06-13。

## 0. 一行で

**ファミリー = 採寸空間の中で「骨格比率を共有し、周径方向にだけ §9 の包絡内で散らばる」点の集合。** 同一ファミリー内は着回し(Merge Armature + 貫通修正)で足り、ファミリーを跨ぐとリターゲティング = スコープ外。

## 1. 問い: いつ着回しはリターゲティング不要か

代表的なアバター向けの衣装を別のアバターに着せたい(所有資産の横断再利用)。体型差が小さければ Merge Armature で着せて残差の貫通を修正すれば足りる。大きければ体型変換(リターゲティング)が要り、それは先行ツールの領分 = 本プロジェクトのスコープ外([DESIGN.md](DESIGN.md))。この境界を、曖昧な「小さい/大きい」でなく**測れる量**として定義したい。

## 2. ファミリーの軸は頭身(比率)、身長ではない

VRChat では身長はゲーム内/ユーザー側で自由にスケールされるので、絶対身長はフィットの判断材料にならない(全体を一様スケールしても貫通分布は変わらない)。残るのは「全体スケールを正規化した後の比率」= **頭身**が捉えるもの。小柄系と標準頭身の差は身長差ではなく比率差で、ここが本質。

頭身は §5 の採寸パス(素体メッシュをスライスする処理)で算出できる**一スカラー**なので、ファミリー判定は機械的に出せる — 人が手で「これは小」とラベル付けする必要はない。

## 3. 近さの3層

1. **同一素体メッシュ** — トポロジ同一・部分スケール差のみ。衣装はその素体に作られているので着回しはほぼ自明、貫通最小。**最内核**
2. **同一頭身ファミリー・別素体** — Humanoid 標準ボーン(VRC ではほぼ全アバター)なら Merge Armature が骨名で通り、残差は中程度だが多くは包絡内。骨マージが効き、貫通修正が残差を吸収
3. **頭身ファミリー跨ぎ** — 比率が根本的に違う。骨マージで深い貫通/分布の崩れ → 包絡外 → **リターゲティング領域 = スコープ外**

「同一ファミリー = 変換ほぼ不要 / 跨ぎ = 変換」は層2と3の境界。層1は最も確実に緑になる核。

## 4. 採寸空間とその計量

各アバター = 周径スカラー十数個(+頭身)のベクトル = 1点(§5)。だがファミリーを決める「距離」は普通のユークリッド距離ではなく、**診断の式そのもの**:

- **部位ごと**(renderer/部位別)
- **半径正規化**(§9 の局所半径 10〜15%。細い手首の 1cm と太ももの 1cm は別物)
- **片側**(body > 衣装設計 の部位だけが貫通。body < 衣装 はゆるみ = LOOSE 側で、距離としては 0)

つまりファミリー所属を決めるのは汎用クラスタリングではなく **§9 の包絡が誘導する関係**。

## 5. 硬い軸と柔らかい軸 ← 核心

なぜ空間が割れるか。**ソルバが半径方向にしか動かさないから**(カプセル SDF 勾配方向の押し出し + 変位場平滑化。[DEFORMATION_METHODS.md] 参照)。半径押し出しは胴を伸ばせない・肩幅を詰められない — 接線方向(表面に沿って材料を再配分する)変形はソルバの射程外。ゆえに軸は2種に分かれる:

- **硬い軸 = 骨格比率(頭身・四肢長比・肩幅対腰幅…)** → 差は接線方向 → ソルバが触れない → **ファミリーの壁**(= §9 の接線限界)。跨ぐ = スコープ外
- **柔らかい軸 = 各部位の周径/半径** → 差は半径方向 → ソルバが ≤1cm 吸収 → **ファミリー内の広がり**。サイズタイリング(S/M/L)はここに住む

**ファミリーの厳密な定義 = 「硬い軸の座標を共有し、柔らかい軸でだけ §9 包絡内に散らばる点の集合」。** これは §9 の「半径方向 vs 接線方向」を空間の軸へ射影しただけで、同じ一本の原理。

## 6. クラスタは「分割」でなく「有向グラフ」

§4 の片側性が効く。貫通は body > 衣装設計 の側にしか出ないので関係は**非対称**:

- A 用の衣装 → やや大きい B = 軽い貫通(包絡内かも)
- A 用の衣装 → やや小さい B = ゆるいだけ(貫通 0、LOOSE 側)

だから「A→B 可」≠「B→A 可」。真の構造は**有向グラフ**で、小/中/大という綺麗な分割はその**損失あり要約**。LOOSE 側([ECOSYSTEM_VISION.md](ECOSYSTEM_VISION.md) §3 のクリアランス統計)はこのグラフの逆向きの辺。

## 7. クラスタは市場が作る — 代表的なアバター = 事実上の重心

採寸空間は一様には埋まらない。点は**代表的なアバターの周りに密集**する(派生アバターが素体を共有し、衣装が代表的なアバターに合わせて作られるため)。だからクラスタリングは市場が既に済ませてあり、**代表的なアバターが事実上の重心**。名前付きファミリー(小/中/大)は「最も近い代表的なアバターはどれか」とほぼ等価。[ECOSYSTEM_VISION.md](ECOSYSTEM_VISION.md) §4 の卒業コホートは「重心は残るが新作供給が止まったクラスタ」と言い換えられる。

## 8. 差分診断 — ファミリー所属の数値テスト

両方のアバターを持つとき、**代表的なアバターは「ついで」ではなく基準線**になる:

- 衣装を本来の代表的なアバターに着せて測る = 設計が意図したフィット ≒ 基準線(緑)
- 同じ衣装を別アバターに着せて測る = 着回しの貫通量
- **差分 = この着回しが設計フィットからどれだけ押し出したか**

差分が小さい = 同一ファミリー、大きい = 跨ぎ。**差分診断 = ファミリー所属の数値テスト**。これはグローバルしきい値の確定を待たずに意味を持つ — 衣装ごとの設計マージン(ゆるい服・きつい服)に自動で追従するため、絶対しきい値が未較正でも「本来0、別体で +4mm」は自明に意味がある。

実装も軽い。`PreflightDiagnostic` を2回回して引くだけ(衣装メッシュは同一、体だけ違うので差分は体型寄与をきれいに分離する)。新しい数学はほぼ要らない、計測プロトコルの追加。

## 9. ラベルは予測、診断が判定

小/中/大は安い事前フィルタとして優秀だが**最終判定はできない**。同じ「中」でも、片方が豊満・くびれ強、もう片方がフラットなら、胸用トップスの着回しは胸で深く貫通する。頭身が同じでも部位比率が違う。

役割分担はこう: **ラベルが「試す価値があるか」を予測し**(同じ小同士ならいける/小→中なら要変換)、**部位別(renderer 別)の差分診断が実際の合否を出す**。ラベルが見落とす部位ミスマッチを診断が拾う。**ラベルは入口、診断が門番。**

## 10. モデルの限界

- **1アバター = 1点でなく領域(ブロブ)** — シェイプキーで胸/腰を動かせるので、所属はユーザー設定に依存する。採寸は計測条件固定(デフォルト・シェイプキー 0; [ECOSYSTEM_VISION.md](ECOSYSTEM_VISION.md) §6)で点を確定させる必要がある
- **境界は構造的に曖昧** — 背の高い小・低い中は所属が割れる。クラスタは境界で損失ありと認め、門番は診断に委ねる(§9)
- **柔らかい軸も大振幅では硬くなる** — 極端な周径差は最終的に接線再配分が要る。§9 の包絡がその切れ目

## 11. 使い道

1. **事前フィルタ** — 最近傍の代表的なアバターで「試す/やめる」を診断の前に出す
2. **タイリングの座る場所** — S/M/L はファミリー内の柔らかい軸上のビン。硬い軸は跨げない壁([ECOSYSTEM_VISION.md](ECOSYSTEM_VISION.md) §4)
3. **Fit Report のシャードキー** — 報告書は (ファミリー, 衣装) で索引。同ファミリーの所有者は同ファミリーの報告だけ見ればよい = ボトムアップ標準の自然な索引

# 変形手法の検討メモ — 貫通修正ソルバの発展候補

このドキュメントは**検討資料**です。現行ソルバ(変位場 Laplacian 平滑化)の限界と、その先の候補手法を比較します。採用が決まった手法は [DESIGN.md](DESIGN.md) の設計制約と [ROADMAP](../ROADMAP.md) に昇格させ、本書は検討の記録として残します。最終更新: 2026-06-12。

## 0. 現状と限界

現行ソルバ(DESIGN.md §6): 検出 → SDF 勾配方向の押し出しを**変位場**に記録 → 変位場への Laplacian 平滑化 →[再検出 → 再押し出し]を 2〜3 回反復。

変位場方式により「修正領域内の衣装ディテールが潰れない」「移動量が侵入深度のオーダーで抑えられる」は達成済み。残る限界は拡散という操作自体に由来する:

- **再沈み込み** — 拡散は押し出し済み頂点の変位も均すため、平滑化のたびに侵入側へ戻り、再検出→再押し出しの反復とそのチューニング(λ・反復数・パス数)が必要になる
- **布の物性を知らない** — 「伸びない」「曲がりにくい」という布の性質を表現できず、裾を持ち上げてドレープし直すべき場面でも布をゴムのように引き伸ばして逃がす
- **回転に弱い** — 変位の線形ブレンドなので、修正が局所回転を伴うべき場面(関節まわりの巻き込み等)でディテールが線形化歪みする

## 1. 評価軸(このプロジェクト固有の制約)

| 軸 | 内容 |
|---|---|
| 実装可能性 | 外部ライブラリなしの C# で書けるか(Unity エディタ内、疎行列ライブラリなし前提) |
| 実行形態 | エディタでのワンショット実行。リアルタイム不要、100ms〜数秒は許容 |

| 保証 | 「最終状態は侵入ゼロ」の構成的保証を維持できるか(最後を押し出し/衝突投影で終えられるか) |

## 2. 候補手法

### 2.1 変位場の調和/重調和補間 — 短期の本命

- **何をするか**: 押し出された頂点では変位 d を固定(Dirichlet 境界条件)、修正領域の外周では d=0 に固定し、その間を Δd=0(調和)または Δ²d=0(重調和)を満たすように線形方程式で解く。「ラプラシアン変形」を座標ではなく変位場に適用したものに相当し、現行の反復拡散はこの近似になっている
- **何が良くなるか**: 押し出し頂点が境界条件になるため**再沈み込みが原理的に消える**。λ・反復数・パス数のチューニングがほぼ不要になり、重調和なら領域境界で接線連続(滑らかな立ち上がり)
- **実装コスト**: 修正領域は小さい(数百〜数千頂点)ので、共役勾配法(CG)を 100 行程度手書きすれば外部依存なしで解ける。現行コードベースの流儀と相性が良い
- **さらに安い応急処置**: Taubin λ|μ 平滑化(正負ステップを交互にかける帯域通過版)。数行の変更で変位の「溶け戻り」をかなり抑えられるが、根本解決は補間化

### 2.2 XPBD 拘束投影 — 高品質モードの本命

- **何をするか**: 3 種の拘束 — 伸び(元のエッジ長を保つ)・曲げ(隣接三角形の二面角を保つ)・衝突(カプセル SDF ≥ マージン)— を Gauss-Seidel で繰り返し投影し、準静的に解く。XPBD は PBD の「反復回数で硬さが変わる」欠点を直した改良版で、硬さをコンプライアンス値として陽に指定できる
- **何が良くなるか**: **布の物性が入る**。調和補間は「修正をどう滑らかに分布させるか」しか知らないが、XPBD はスカートの裾を太ももが押す場面で裾全体を持ち上げて「ドレープし直す」方向に動く
- **実装コスト**: 線形ソルバ不要・SVD 不要・依存ゼロのループだけで書ける。実装難度に対して結果が良いことで知られる。最後を衝突投影パスで終えれば「最終状態は侵入ゼロ」の構成的保証もそのまま引き継げる
- **パラメータ**: stretch/bend のコンプライアンス、反復数。服種ごとのプリセット化と相性が良い

### 2.3 ARAP(As-Rigid-As-Possible)— 回転を伴う変形でのディテール保持

- **何をするか**: 頂点(セル)ごとに最適回転を SVD/極分解で推定し、大域の線形解(cotangent Laplacian、前因子分解可)と交互に反復する
- **何が良くなるか**: ラプラシアン変形の弱点(微分座標が回転しないため、大きく曲がる変形でディテールが歪む)を回転推定で解消。硬い装飾・アーマー・厚手コートなど「形を剛体的に保ってほしい」服に強い
- **実装コスト**: 本書の候補中で最重量(cotangent 重み+疎行列の因子分解+セルごとの SVD)。XPBD で不足が見えてからで遅くない。ROADMAP フェーズ4「高品質モード」に既載

### 2.4 RBF 空間変形場 — VRC 衣装の現実への保険

- **何をするか**: 押し出し修正量を制御点として RBF 補間場を構築し、メッシュではなく**空間そのもの**を変形する。グラフ(三角形の繋がり)に依存しない別系統
- **何が良くなるか**:
  - **浮遊パーツに唯一強い** — ボタン、リボン、布から浮いた装飾はグラフベースの手法では繋がっていないため追従しない(現行の溶接は同一位置頂点のみ)。空間場なら袖の修正にボタンが自然に追従する
  - **重ね着の修正一貫性** — 現行ソルバはレンダラーごとに独立に解くため、シャツだけ押し出してジャケットを突き抜ける事態が起こりうる。単一の空間場を全レンダラーに適用すれば修正の一貫性が保てる
  - No Cache と両立(場はセッション内で構築して捨てる。プロファイル保存とは無関係)
- **実装コスト**: 制御点数 k に対して O(k³) の線形解(押し出し頂点のサブサンプリングで k を抑える)。カーネルと半径の選定が要る

## 3. 見送り

- **IPC(Incremental Potential Contact)** — barrier 法による交差ゼロ保証付き接触解法。品質は最強だが Newton 法+直線探索の実装・計算コストがエディタツールには過剰
- **Projective Dynamics** — XPBD と全体最適の中間。XPBD の収束に不満が出た場合の代替として記憶に留める
- **学習ベース(ニューラル変形)** — データ要件が重く、挙動の監査可能性・No Cache の思想とも不整合

## 4. 推奨の階段

```
現行(変位場 Laplacian)
  → ① 調和補間(CG 手書き、再沈み込み根絶)     … 同じ枠組みの完成形
  → ② XPBD(伸び・曲げ・衝突拘束)             … 「布らしさ」の獲得
  → ③ ARAP(必要になったら)                   … 剛体的ディテール保持
  ④ RBF 空間場は独立に小さく試作する価値あり    … 浮遊パーツ・重ね着は①〜③では解けない
```

服種で最適解は異なる: タイト系(ボディスーツ)=現行の射影で十分/ドレープ系(スカート・ワンピース)= XPBD/硬質系(アーマー・装飾)= ARAP。最終的にはフェーズ3のマスクペイントと組み合わせて「部位・服ごとに方式を選ぶ」設計に収束させるのが自然。

## 5. どこを直すか(適用範囲の考え方)

手法(どう直すか)と直交する軸として、適用範囲(どこを直すか)の整理:

- **静的貫通に絞り込み機能は不要** — 検出ベースなので侵入がある所しか触らない。絞るまでもなく絞られている
- **関節(肘・肩・膝)はポーズ依存の別レイヤー** — 素体と衣装のウェイト塗りの差 × 関節回転角で生じるため、**体型差がゼロでも貫通する**。本命はフェーズ3「代表ポーズ対応」(複数ポーズで解いて結果を束ねる)
- **安価な中間策: 関節部の可変マージン** — 関節をまたぐウェイトを持つ頂点だけマージンを 1.5〜2 倍にし、曲げ起因の貫通への保険とする(ROADMAP フェーズ3に追加済み)
- **任意ポーズでの修正は現状でも可能** — キャプチャ(BakeMesh)と書き戻し(逆スキニング)がポーズ対応のため、エディタで関節を曲げてから Run すればそのポーズで直せる。ただし結果はそのポーズに最適化される点に留意
- **素体削除運用との関係** — 完全に服で隠れる部位は素体ポリゴン削除が正解で、本ツールと競合しない。主たる対象は**削除できない肌見せ境界**(袖口・襟元・裾・ソックス上端)であり、それは構造的に関節の近くに集中する(袖口⇔肘、裾⇔膝)

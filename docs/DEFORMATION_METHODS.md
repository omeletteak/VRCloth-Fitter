# 変形手法の検討メモ — 貫通修正ソルバの発展候補

このドキュメントは**検討資料**です。現行ソルバ(変位場 Laplacian 平滑化)の限界と、その先の候補手法を「**着せた後の貫通修正**」という問題設定に照らして比較します。採用が決まった手法は [DESIGN.md](DESIGN.md) の設計制約と [ROADMAP](../ROADMAP.md) に昇格させ、本書は検討の記録として残します。最終更新: 2026-06-17。

## 0. 現状と限界

現行ソルバ(DESIGN.md §6): 検出 → SDF 勾配方向の押し出しを**変位場**に記録 → 変位場への Laplacian 平滑化 →[再検出 → 再押し出し]を 2〜3 回反復。

変位場方式により「修正領域内の衣装ディテールが潰れない」「移動量が侵入深度のオーダーで抑えられる」は達成済み。残る限界の本丸は、現行が変位場を **Vector3 全成分まとめて等方平滑化している**ことに由来する:

- **再沈み込み(核心)** — 等方平滑化は押し出した**法線方向の高さ**も周囲へ拡散させる。押し出し頂点(変位大)と未押し出しの周囲(変位ゼロ)を均すと押し出し量が削れ、侵入側へ戻る。これを補うために再検出→再押し出しの反復と、そのチューニング(λ・反復数・パス数)が必要になっている
- **レンダラー間の不整合** — 現行はレンダラーごとに独立に解くため、シャツだけ押し出してジャケットを突き抜ける事態が起こりうる
- **浮遊パーツの非追従** — グラフ(三角形の繋がり)ベースのため、布から浮いた装飾(ボタン・リボン)は袖の修正に追従しない(現行の溶接は同一位置頂点のみ)

## 1. 「着せた後の貫通修正」の本質(評価の物差し)

候補手法を評価する前に、この問題が**体型変換(リターゲティング)とは別物**だと明確にする。両者を混同すると、リターゲ向けの重い大変形手法を不要に担ぐことになる。貫通修正に固有の性質:

1. **変位が小さい** — 侵入深度のオーダー(数 mm〜数 cm)。元の衣装デザインを尊重し、布の大域形状は変えない
2. **方向がほぼ確定している** — 押し出す向きは SDF 勾配(外向き法線)で決まっており、「どう変形させるか」の探索自由度がリターゲより圧倒的に低い
3. **回転がほとんど起きない** — 2 の帰結。大きく曲がる変形を伴わないため、回転推定を売りにする手法は空振りする
4. **侵入ゼロを構成的に保証したい** — 最後は必ず衝突投影で終える

→ この 4 点が、§3 の採用と §4 の降格を分ける基準になる。

## 3. 採用候補(貫通修正の本質に適合する手法)

### 3.1 法線/接線分解 + 反復投影 — 短期の本命

- **何をするか**: 変位を「法線成分(SDF が決める表面+マージンの高さ)」と「接線成分(横方向の段差)」に分け、**法線成分は平滑化せず毎反復スナップし直し、接線成分だけを平滑化する**。現行の `LaplacianSmoothing.Smooth`(全成分均し)を成分分解に差し替えるだけの小改修
- **何が良くなるか**: 法線方向は毎反復で衝突拘束を満たし直すので、**再沈み込みが原理的に消える**(§0 の核心を直接潰す)。線形ソルバを書かずに λ・反復数・パス数のチューニング地獄から抜けられる
- **位置づけ**: これを反復粒度で書くと「平滑化 1 ステップ → 侵入頂点を法線方向へ再投影 → 繰り返し」となり、実質 **stretch/bend 拘束を抜いた XPBD の衝突拘束版**になる。§3.2・§3.3 はこの自然な延長線上にある

### 3.2 変位場の調和/重調和補間 — 滑らかさが不足したときの昇格先

- **何をするか**: 押し出された頂点では変位 d を固定(Dirichlet 境界条件)、修正領域の外周では d=0 に固定し、その間を Δd=0(調和)または Δ²d=0(重調和)を満たすよう線形方程式で解く。現行の反復拡散はこの近似であり、§3.1 はその「法線方向だけ厳密に固定した」軽量版にあたる
- **何が良くなるか**: 押し出し頂点が境界条件になるため再沈み込みが消え、重調和なら領域境界で接線連続(滑らかな立ち上がり)。§3.1 で段差の馴染みが足りない場合の上位互換
- **実装コスト**: 修正領域は小さい(数百〜数千頂点)ので、共役勾配法(CG)を 100 行程度手書きすれば外部依存なしで解ける

### 3.3 軽量 XPBD(衝突 + stretch 拘束)— 「布らしさ」が要るときだけ

- **何をするか**: §3.1 に **stretch 拘束(元のエッジ長を保つ)**を 1 本足す。衝突投影と交互に Gauss-Seidel で解く
- **何が良くなるか**: スカートの裾を太ももが押す場面で、裾を引き伸ばさず全体を持ち上げて「ドレープし直す」方向に動く。bend 拘束まで足せばさらに布らしくなるが、貫通修正の主戦場では stretch だけで足りる見込み
- **実装コスト**: 線形ソルバ不要・依存ゼロのループのみ。最後を衝突投影で終えれば侵入ゼロ保証も維持

### 3.4 レンダラー間・浮遊パーツの一貫性 — 重い手法を待たずに

- **重ね着**: §0 の不整合は、まず**全レンダラーを統合した 1 つの SDF / 位置配列で解く**だけで大半が片付く(現行がレンダラーごと独立に解くのが根因)。空間変形場(§4.2)を担ぐ前に試す
- **浮遊パーツ**: 溶接対象を「同一位置頂点」から「近接頂点クラスタ」へ広げる、または親三角形に剛体追従させる軽量策で多くは足りる

## 4. 降格・隔離(リターゲ=大変形向けで、貫通修正には過剰)

§1 の本質に照らすと、以下は貫通修正の主経路から外す。必要が**実機 E2E で実証されてから**、それぞれの隔離先で扱う。

### 4.1 ARAP(As-Rigid-As-Possible)→ フェーズ4「高品質モード」へ隔離

頂点ごとに最適回転を SVD/極分解で推定し、大域線形解と交互反復する手法。「大きく曲がる変形でのディテール保持」が売りだが、貫通修正は**小変位でほとんど回転を伴わない(§1-3)**ため回転推定がほぼ空振りする。本書候補中で最重量(cotangent 重み+疎行列因子分解+セルごと SVD)。剛体的ディテール保持が要る硬質系の服が実機で問題になったときだけ、ROADMAP フェーズ4 で検討。

### 4.2 RBF 空間変形場 → 必要が出たら独立に小さく試作

押し出し修正量を制御点として RBF 補間場を構築し、メッシュではなく空間そのものを変形する別系統。**変換型ツールが体型変換に用いる空間変形場と同系統**であり、貫通修正の小変位という問題設定にはオーバースペック(制御点数 k に対し O(k³) の線形解)。唯一の固有利点は浮遊パーツへの追従だが、それは §3.4 の軽量策でまず代替を試す。重ね着・浮遊パーツが §3.4 で解けないと実証された場合に限り、独立試作の価値あり(No Cache とは両立 — 場はセッション内で構築して捨てる)。

### 4.3 見送り(参考)

- **IPC(Incremental Potential Contact)** — barrier 法による交差ゼロ保証付き接触解法。品質は最強だが Newton 法+直線探索の実装・計算コストがエディタツールには過剰
- **Projective Dynamics** — XPBD と全体最適の中間。XPBD の収束に不満が出た場合の代替として記憶に留める
- **学習ベース(ニューラル変形)** — データ要件が重く、挙動の監査可能性・No Cache の思想とも不整合

## 5. 推奨の階段

```
現行(変位場 等方Laplacian)
  → ① 法線/接線分解 + 反復投影     … 現行の小改修。再沈み込み根絶。まずこれ
  → ② 調和/重調和補間(CG手書き)   … ①で滑らかさが不足したら昇格
  → ③ 軽量XPBD(stretch 1本)      … スカート裾の「布らしさ」が要るときだけ
  ─────────────────────────────
  重ね着・浮遊パーツ … まず §3.4 の軽量策(統合SDF・近接溶接)で試す
  ✗ ARAP          … 回転がほぼ無く空振り。フェーズ4「高品質モード」へ隔離(§4.1)
  △ RBF空間場     … 実機で §3.4 が不足と実証されたら独立試作(§4.2)
```

服種で最適解は異なる: タイト系(ボディスーツ)=①の射影で十分/ドレープ系(スカート・ワンピース)=③/硬質系(アーマー・装飾)=ARAP。最終的にはフェーズ3のマスクペイントと組み合わせて「部位・服ごとに方式を選ぶ」設計に収束させるのが自然。

## 6. どこを直すか(適用範囲の考え方)

手法(どう直すか)と直交する軸として、適用範囲(どこを直すか)の整理:

- **静的貫通に絞り込み機能は不要** — 検出ベースなので侵入がある所しか触らない。絞るまでもなく絞られている
- **関節(肘・肩・膝)はポーズ依存の別レイヤー** — 素体と衣装のウェイト塗りの差 × 関節回転角で生じるため、**体型差がゼロでも貫通する**。本命はフェーズ3「代表ポーズ対応」(複数ポーズで解いて結果を束ねる)
- **安価な中間策: 関節部の可変マージン** — 関節をまたぐウェイトを持つ頂点だけマージンを 1.5〜2 倍にし、曲げ起因の貫通への保険とする(ROADMAP フェーズ3に追加済み)
- **任意ポーズでの修正は現状でも可能** — キャプチャ(BakeMesh)と書き戻し(逆スキニング)がポーズ対応のため、エディタで関節を曲げてから Run すればそのポーズで直せる。ただし結果はそのポーズに最適化される点に留意
- **素体削除運用との関係** — 完全に服で隠れる部位は素体ポリゴン削除が正解で、本ツールと競合しない。主たる対象は**削除できない肌見せ境界**(袖口・襟元・裾・ソックス上端)であり、それは構造的に関節の近くに集中する(袖口⇔肘、裾⇔膝)

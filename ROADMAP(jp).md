# VRCloth-Fitter 開発ロードマップ

このドキュメントは、VRCloth-Fitterの開発計画と機能の履歴を概説します。

## ゴール

このツールの目標は、アバターの衣装カスタマイズのための堅牢でコミュニティ主導の機能群を提供し、非破壊的なワークフローを保証することです。

### フェーズ1：基本的なスケールフィッティング（バックエンド）
- [x] **スケール計算**
- [x] **スケーリングの適用（NDMF Pass）**

### フェーズ2：高度なメッシュ変形（バックエンド）
- [x] **変形データ用コンポーネント**
- [x] **メッシュ変形アルゴリズム**
- [x] **非破壊的な適用（NDMF Pass）**

### フェーズ3：機能拡張（バックエンド＆ロジック）
- [x] **ブレンドシェイプ同期ロジック**
- [x] **マテリアル＆シェーダーユーティリティのロジック**
- [x] **UI日本語対応システム**

### フェーズ4：コミュニティ機能
- [x] **プリセットのインポート/エクスポート**:
    - [x] スケール調整およびメッシュ変形のデータをJSONファイルとしてエクスポート/インポートする機能を実装する。
- [ ] **GUIDによるプリセットシステムの改善**:
    - [ ] エクスポートされるプリセットファイルに、元のアバターのPrefab GUIDを記録する。
    - [ ] インポート時に、プロジェクト内の一致するGUIDを検索し、対象アバターを自動検出する。
    - [ ] GUIDが見つからない場合、アバター名で検索するフォールバック機能を実装する。

### フェーズ5：UI/UXの全面的な再設計（コンポーネントベースのワークフロー）
- [ ] **メインコンポーネントの作成 (`VRClothFitter.cs`)**:
    - [ ] ユーザーが衣装オブジェクトに追加するための、新しいメインコンポーネントを開発する。このコンポーネントが対象アバターへの参照を保持する。
- [ ] **カスタムエディタの作成 (`VRClothFitterEditor.cs`)**:
    - [ ] 古い`VRClothFitterWindow`のすべてのUIとロジックを、`VRClothFitter`コンポーネント用の新しいカスタムエディタに移行する。
    - [ ] これにより、Modular Avatarのような、より直感的でインスペクタベースのワークフローを提供する。
- [ ] **全機能のカスタムエディタへの統合**:
    - [ ] **ボーンマッピングUI**: 新しいカスタムエディタ内にボーンマッピングのインターフェースを再実装する。
    - [ ] **ブレンドシェイプ同期UI**: ブレンドシェイプ同期のインターフェースを再実装する。
    - [ ] **マテリアルユーティリティUI**: マテリアル変換ユーティリティを再実装する。
    - [ ] **プレビュー機能**: リアルタイムプレビューボタンを統合する。
- [ ] **古いエディタウィンドウの廃止**:
    - [ ] `[MenuItem]`属性を削除して古いウィンドウを非表示にし、最終的に`VRClothFitterWindow.cs`ファイルを段階的に廃止する。

---

*このロードマップは、コミュニティからのフィードバックや開発の進捗に応じて変更される可能性があります。*

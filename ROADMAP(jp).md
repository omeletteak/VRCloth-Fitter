# VRCloth-Fitter 新ロードマップ（改訂版）

このドキュメントは、VRCloth-Fitterの開発計画と機能拡張の方針を示します。

## 基本理念

- **Modular Avatar (MA)** のエコシステムと完全互換、非破壊ワークフローを徹底する
- **Merge Armature** による骨格揃えを前提に、その後の「衣装メッシュの貫通修正」を自動化
- 計算済みの変形を **差分キャッシュ** として保存し、再利用可能にすることでユーザー負担を最小化
- **差分キャッシュの安全な共有**: 変換元アバターの著作権を保護しつつ、ユーザー間で補正データを共有できる仕組みを構築
- 全機能を **MITライセンス・オープンソース** で公開し、コミュニティ貢献と透明性を重視

---

## フェーズ1：基盤構築（必須ワークフロー）

- [x] **Merge Armature連携**  
  MAのMerge Armature API/処理後にフックし、衣装とアバターが骨格的に一致した状態から処理を開始

- [x] **プロキシボディ生成**  
  ボーンカプセル／低ポリメッシュを自動生成し、貫通検出用の簡易コリジョンとして利用

- [ ] **貫通検出**  
  - 各衣装頂点の最近接点と距離（signed distance）を計算  
  - 閾値以下を「侵入頂点」としてマーク

- [ ] **変形差分適用（軽量版）**  
  - 頂点法線方向＋投影ベクトルのハイブリッドで押し出し  
  - ラプラシアン平滑化で形状を自然に保持  
  - 差分マップをScriptableObject/JSONとして保存（非破壊）

---

## フェーズ2：モード別パイプライン実装

- [ ] **セキュア共有モード（変換元アバター不要）**
  - カプセルプロキシ等の汎用人体モデルを基準に変形
  - 差分を局所化・量子化・ノイズ付加することで、元アバターの形状復元を不可能にする
  - 低精度だが安全に共有可能な補正データとして利用

- [ ] **中品質モード（両アバター必須）**
  - Merge Armature → Capsule Proxy → 局所ARAP → Laplacian smoothing
  - 多くのケースで自然な見た目と精度を両立

- [ ] **高品質モード（両アバター必須）**
  - Merge Armature → NRRまたは全域ARAP → マルチポーズ最適化 → Laplacian smoothing
  - 最終出力向け、数秒〜数分の計算時間想定

---

## フェーズ3：差分キャッシュ最適化

- [ ] **差分キャッシュ管理UI**  
  - 計算済みペア（衣装・アバター）の一覧表示  
  - 差分適用、更新、削除、共有モードへの変換をGUIで操作

- [ ] **部分適用・合成**  
  - 上半身・下半身・腕など部位ごとに差分を保存  
  - 別アバターや別衣装へ流用可能にする

---

## フェーズ4：差分キャッシュのセキュア共有

- [ ] **差分安全化処理**
  - 差分情報を汎用人体モデルからの相対変位に変換（局所化）
  - 座標の量子化とノイズ付加により、元形状の復元を困難にする
  - 安全化済み差分のプレビュー機能

- [ ] **共有プリセット**
  - 安全化された差分キャッシュをファイルとしてエクスポート
  - 他ユーザーがインポートして、自身の別アバターへ軽量補正を適用

---

## フェーズ5：ユーザビリティ向上

- [ ] **シーンビュー可視化**  
  - 侵入頂点をヒートマップ表示  
  - 修正結果のBefore/After比較

- [ ] **マスクペイント**  
  - 修正したくない部位をペイントで除外

- [ ] **プリセット保存・共有**  
  - 差分やパイプライン構成をファイル化し、コミュニティで配布可能

---

## フェーズ6：高度機能・将来展望

- [ ] **FFDケージ編集モード**  
  - ユーザーがラティスを操作して微調整可能  
  - 結果をブレンドシェイプとして保存

- [ ] **プロジェクション（投影）修正**  
  - ケージ変形後や自動修正後の微細な貫通を、シュリンクラップやコンフォームで補正

- [ ] **AI/MLベース補正（研究枠）**  
  - 学習済みモデルによる自動フィット提案  
  - 差分キャッシュとの組み合わせで未知ペアにも即適用

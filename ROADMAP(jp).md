# VRCloth-Fitter 開発ロードマップ

このドキュメントは、VRCloth-Fitterの開発計画と機能の履歴を概説します。

## 基本理念

このツールの目標は、アバターの衣装カスタマイズのための、堅牢で直感的な機能群を提供することです。我々の基本理念は、**車輪の再発明を避け**、業界標準である**Modular Avatar (MA)**の強力な機能を可能な限り活用することです。

VRCloth-Fitterの主な役割は、MA自体が提供しない高度な計算や自動化を行い、MAコンポーネントのための**インテリジェントなセットアップユーティリティ**として機能することです。これにより、ユーザーにシームレスかつ非破壊的なワークフローを保証します。

### フェーズ1：概念実証（レガシー）

- [x] **初期実装**: 独自のNDMF Passを持つ、スタンドアロンのウィンドウベースツールを開発。このフェーズで、中核となるフィッティングアルゴリズムを検証しました。

### フェーズ2：アーキテクチャの再設計 - Modular Avatarとの相乗効果

このフェーズは、Modular Avatarと深く連携し、中核機能をMAに委譲することで、安定性とシンプルさを最大化するための、根本的なアーキテクチャの転換を示します。

- [x] **UI/UXの全面的な見直し**:
    - [x] エディタウィンドウから、モダンで直感的なコンポーネントベースのワークフロー（`VRClothFitter`コンポーネントとカスタムエディタ）へ移行しました。
- [ ] **独自スケーリングシステムの廃止**:
    - [ ] **`ScalingPass.cs`と`VRClothFitterScalingData.cs`を削除**。ビルド時のスケーリングは、`MA Merge Armature`に完全に委譲します。
- [ ] **「スケール計算」機能の再定義**:
    - [ ] 「スケール計算」ボタンは、もはや独自コンポーネントにデータを保存しません。新しい役割は以下の通りです：
        1.  アバターと衣装の素体アーマチュア間の**プロポーションの違い**（ボーンの長さや太さ）を計算する。
        2.  計算したスケール比を、一度限りのセットアップ処理として、**衣装プレハブのボーンのTransformに直接適用**する。
- [ ] **`MA Merge Armature`セットアップの自動化**:
    - [ ] 「スケール計算」ボタンが、衣装に`MA Merge Armature`コンポーネントを自動的に追加するようにする。
    - [ ] `Match Bone Scale`オプションを有効化し、MAが将来的なスケーリング調整（例：アバターの身長スライダーなど）をビルド時に非破壊的に処理できるようにする。
- [ ] **`Fit Bones`機能の廃止**:
    - [ ] 手動でのボーンの親子付けは`MA Merge Armature`に完全に委譲されるため、`Fit Bones`ボタンは廃止します。

### フェーズ3：高度なフィッティング機能

これらの機能は、Modular Avatarがネイティブで提供するもの以上の価値を提供します。

- [x] **高度なメッシュ変形**:
    - [x] コントロールアンカーを使用して衣装メッシュを直接変形させ、特にプロポーションが大きく異なる場合（例：下半身）の複雑なフィッティング問題を解決するシステム。これはVRCloth-Fitterの中核的かつユニークな機能として維持されます。
- [x] **ブレンドシェイプ同期ヘルパー**:
    - [x] ブレンドシェイプを簡単に対応付け、`MA Blendshape Sync`コンポーネントを自動設定するUI。
- [x] **マテリアル＆シェーダーユーティリティ**:
    - [x] マテリアルを指定のシェーダーに一括変換するユーティリティ。

### フェーズ4：コミュニティ機能と将来の展望

- [x] **プリセットのインポート/エクスポート**:
    - [x] フィッティング設定（メッシュ変形アンカーなど）をJSONファイルとして保存・読み込みするシステム。
- [ ] **GUIDによるプリセットシステムの改善**:
    - [ ] アバターのPrefab GUIDを利用して、より信頼性の高いアバター自動検出機能を実装する。

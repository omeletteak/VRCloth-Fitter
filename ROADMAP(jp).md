# VRCloth-Fitter 開発ロードマップ

このドキュメントは、VRCloth-Fitterの開発計画と機能の履歴を概説します。

## 基本理念

このツールの目標は、アバターの衣装カスタマイズのための、堅牢で直感的な機能群を提供することです。我々の基本理念は、**車輪の再発明を避け**、業界標準である**Modular Avatar (MA)**の強力な機能を可能な限り活用することです。

VRCloth-Fitterの主な役割は、MA自体が提供しない高度な機能を提供し、MAコンポーネントのための**インテリジェントなセットアップユーティリティ**として機能することです。これにより、ユーザーにシームレスかつ非破壊的なワークフローを保証します。

---

### フェーズ1 & 2：レガシー（ボーンベースのフィッティング）

- [x] **初期実装**: プロポーショナルなボーンスケーリングと、アンカーベースのメッシュ変形システムを開発。
- [x] **アーキテクチャの再設計**: Modular Avatarのフックと深く連携し、非破壊的なビルドプロセスを実現。
- **結論**: ボーンスケーリングは機能したものの、メッシュ変形のアプローチは不安定かつ根本的に欠陥があり、メッシュの収縮を引き起こすことが頻発。このアプローチは現在、**廃止**と見なされます。

---

## フェーズ3：新しい基盤 - ケージ変形

このフェーズは、フィッティング戦略の全面的な見直しを示します。プログラムによる自動変形から、ユーザー主導のインタラクティブなアプローチへと移行します。目標は、`MA Merge Armature`が対応しない**体型の違い**の問題を解決するツールを提供することです。

- [ ] **コア機能：ケージ（ラティス）変形**
    - [ ] **ケージ生成**: 対象の衣装メッシュの周囲に、設定変更可能な単純なケージ（例：3x3x3の格子）を自動生成する機能を実装する。
    - [ ] **インタラクティブな操作**: ユーザーがシーンビュー上でケージの制御点を選択し、動かせるようにする。
    - [ ] **リアルタイムのメッシュ変形**: ユーザーがケージを操作すると、それに合わせて内部の衣装メッシュが滑らかに変形する。コアロジックは、フリーフォーム変形（FFD）または同様のラティスベースのアルゴリズムに基づく。
    - [ ] **非破壊的なワークフロー**: すべての編集は、一時的なメッシュインスタンスに対して行われる。

- [ ] **コア機能：ブレンドシェイプへのエクスポート**
    - [ ] ケージ変形の結果を、新しい**ブレンドシェイプ**として元のメッシュに保存する機能を実装する。
    - [ ] これにより、全プロセスが非破壊的となり、Unityの標準機能やModular Avatarなどの他ツールと完璧に連携できる。

- [ ] **新しいコンポーネントベースのUI/UX**
    - [ ] 新しい`VRClothFitterCageDeformer` MonoBehaviourを作成する。
    - [ ] このコンポーネントのためのカスタムエディタを設計し、ケージの生成、編集状態の管理（編集開始/終了）、ブレンドシェイプへの保存といった操作を行えるようにする。
    - [ ] 古い`VRClothFitterDeformationData`と関連システムは廃止する。

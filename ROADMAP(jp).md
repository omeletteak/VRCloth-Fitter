# VRCloth-Fitter 開発ロードマップ

このドキュメントは、VRCloth-Fitterの開発計画と機能の履歴を概説します。

## 基本理念

このツールの目標は、アバターの衣装カスタマイズのための、堅牢で直感的な機能群を提供することです。我々の基本理念は、**車輪の再発明を避け**、業界標準である**Modular Avatar (MA)**の強力な機能を可能な限り活用することです。

VRCloth-Fitterの主な役割は、MA自体が提供しない高度な計算や自動化を行い、MAコンポーネントのための**インテリジェントなセットアップユーティリティ**として機能することです。これにより、ユーザーにシームレスかつ非破壊的なワークフローを保証します。

### フェーズ1：概念実証（レガシー）

- [x] **初期実装**: 独自のNDMF Passを持つ、スタンドアロンのウィンドウベースツールを開発。

### フェーズ2：アーキテクチャの再設計 - Modular Avatarとの最終的な連携

このフェーズは、Modular Avatarの拡張ポイントと深く連携し、完全に非破壊的なビルドプロセスを実現しつつ、一時的な破壊的プレビューによって即座の視覚的フィードバックを提供するための、最終的なアーキテクチャを定義します。

- [x] **UI/UXの全面的な見直し**:
    - [x] モダンで直感的なコンポーネントベースのワークフローへ移行しました。
- [x] **`MergeArmatureHook`の採用によるスケーリング**:
    - [x] `MergeArmatureHook`を実装する新しいコンポーネント/スクリプトを作成する。
    - [x] このフックは、ビルド時に`VRClothFitterScalingData`からスケールデータを読み込み、その場でボーンに適用することで、**完全に非破壊的なビルド**を実現する。
- [x] **データコンポーネントの再導入**:
    - [x] 計算されたボーンスケール比率を、プレハブを変更しない純粋なデータとして保存するために`VRClothFitterScalingData.cs`を再導入する。
- [x] **「破壊的プレビュー」システムの実装**:
    - [x] エディタに「プレビュー開始」「プレビュー終了」ボタンを追加する。
    - [x] 「プレビュー開始」は、データコンポーネントのスケール値を一時的に衣装プレハブのボーンに直接適用し、即座の視覚的フィードバックを可能にする。
    - [x] 「プレビュー終了」は、ボーンを元のスケールに戻し、プレハブがクリーンな状態を保つようにする。
- [x] **「スケール計算」機能の再定義**:
    - [x] この機能の役割は、プロポーション差を計算し、その結果を**`VRClothFitterScalingData`コンポーネントに保存する**こととなる。
- [x] **`Fit Bones`機能の廃止**:
    - [x] ボーンの親子付けは`MA Merge Armature`に完全に委譲されます。

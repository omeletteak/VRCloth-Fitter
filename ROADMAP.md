# VRCloth-Fitter ロードマップ

このドキュメントは VRCloth-Fitter の開発計画を示します。背景にある設計判断と競合分析は [docs/DESIGN.md](docs/DESIGN.md) を、ドキュメント全体の索引は [docs/README.md](docs/README.md) を参照してください。

## 基本理念

- **貫通修正に特化する** — 衣装の体型変換(リターゲティング)は先行ツールの領分。本プロジェクトは「着せた後に残る貫通の自動修正」という、誰も埋めていない後段を担う
- **No Cache** — アバター素体形状を復元しうるデータを保存・出力しない。形状由来の中間データはその場で計算し、その場で捨てる
- **Modular Avatar / NDMF エコシステムとの互換** — Merge Armature による骨格揃えを前提とし、非破壊ワークフローを徹底する
- **MIT ライセンスのオープンソース** — 上記の約束(特に No Cache)をコードで検証可能にする

## フェーズ1: 貫通修正コア MVP(進行中)

- [x] **プロキシボディ生成** — Humanoid ボーンからカプセル列を自動生成
- [x] **カプセル距離計算** — 最近接点・符号付き距離・押し出し先の計算(EditMode テスト付き)
- [x] **シーンビュー可視化** — カプセルのワイヤ表示と、侵入頂点の深度ヒートマップ表示
- [x] **パイプラインの実メッシュ配線** — 衣装 SkinnedMeshRenderer の頂点をワールド座標で取得
- [x] **貫通検出** — 全頂点×全カプセルの符号付き距離から侵入頂点をマーク
- [x] **押し出し** — 侵入頂点をカプセル表面+マージンへ移動(方向は SDF 勾配に統一)
- [x] **ラプラシアン平滑化** — 押し出し変位場を修正領域限定で平滑化して段差を馴染ませる(再侵入は再押し出し)
- [x] **非破壊適用** — メッシュ複製への書き戻し(元アセット不変、Undo 対応)
- [x] **プリフライト診断** — 検出統計(最大深度・p95・侵入率・最大連結パッチ・深さ/カプセル半径比)からサポート範囲([docs/DESIGN.md](docs/DESIGN.md) 第9節)内かを緑/黄/赤判定し、赤は適用を既定で中止(しきい値の較正は E2E で)
- [x] **素体表現: メッシュSDF コライダ** — **v1 landed**。衝突バックエンドを `IBodyCollider`(符号付き距離・勾配・局所厚みの3演算)に抽象化し、`MeshSdfCollider`(最近接三角形距離 + 一般化巻き数による符号判定)と `CapsuleBodyCollider`(既存カプセルの合成、数学的に従来挙動と等価)を実装。素体メッシュはセッション内でメモリ構築し捨てる(保存しない=No Cache 両立、[docs/DESIGN.md](docs/DESIGN.md) §6)。巻き数の絶対値判定により非水密メッシュにも堅牢(ユニットテストで穴あきメッシュの内外判定を確認)。検出・押し出し・平滑化・適用・プリフライトは抽象越しに不変。インスペクタの **Use Mesh SDF Collider** トグルで切替(**既定OFF**:実機 E2E でしきい値較正するまでカプセルが既定)。ユニット 89 件緑(合成ヌルテスト+クロスフィット+加速版とブルートフォースの厳密一致を含む)。**性能加速も landed**(2026-06-15): 当初ブルートフォースは実サイズで遅かった(素体 50,400 三角形×衣装 2,009 頂点で 1 スキャン≈16.4 秒・ソルバ全段≈82 秒/レンダラー)が、**BVH 分枝限定(最近接, 厳密)+ Barnes–Hut 巻き数(遠方クラスタをダイポール近似, 表面付近は厳密)**で 1 スキャン≈63 ms(約 260 倍)・ソルバ全段≈315 ms へ短縮。非水密堅牢性は維持。残: **実機 E2E での偽貫通解消の目視確認(手順0b、人手ゲート)** のみ
- [ ] **E2E 検証** — 実アバター+衣装で貫通解消を確認(上記5項目の実機目視をまとめて行う。手順は [docs/E2E_TEST_GUIDE.md](docs/E2E_TEST_GUIDE.md))

## フェーズ3: 品質向上

- [ ] **カプセル半径の自動推定** — 素体メッシュから半径を推定。**v1 landed**(最近接軸割り当て+パーセンタイル+外れ値ゲート; 胴/首/脚/上腕はアバター別の妥当値へ、胴は固定0.12→約0.06で誤検出消滅)。残課題は当初スキニング基準の半径割り当てを想定したが、E2E ヌルテストで**円断面カプセルは胴・足の形状を半径調整では表せない**と判明。解決はフェーズ1の**メッシュSDF コライダ**へ移譲(本項目はその前段の知見として残す)。カプセルは高速プレビュー用に残す可能性
- [ ] **シェイプキー・ポーズ対応** — 代表ポーズ/シェイプキーでの貫通もまとめて修正。あわせて**適用の二重適用を解消する**: 現状の書き戻し([VRClothMeshApplier](Assets/VRCloth-Fitter/Editor/VRClothMeshApplier.cs))は焼いた座標(シェイプキー変位込み)でベース頂点を置換しブレンドシェイプを複製に残すため、非0のシェイプキー(Modular Avatar の Blendshape Sync で素体に追従)が二重に乗って崩れる。**「焼いた座標で置換」ではなく「元ベース頂点へ補正デルタを加算」**に変えれば静的体型では二重適用が消える(動的に変わる Sync 値の追従が本項の本丸)
- [ ] **縮め/隠しシェイプキーへの堅牢性** — 衣装の一部を意図的に縮める/隠すシェイプキー(Blendshape Sync 無しの独立トグル)は、畳んだ頂点を体内深部へ押し込む。検出はこれを深い貫通として正しく拾うが、結果は偽RED→当該レンダラーごと skip になる(安全側だが、同レンダラーの可視部分の実貫通も巻き添えで未修正になりうる)。畳み込み(体内深部で密集・縮退)頂点を検出から除外し、可視部分だけを対象にする。実機 E2E で確認(左ソックス短縮SKが約53mm内側と誤検出、中立化で解消 — [docs/E2E_TEST_GUIDE.md](docs/E2E_TEST_GUIDE.md) §0.1)
- [ ] **関節部の可変マージン** — 関節をまたぐウェイトを持つ頂点のマージンを増し、曲げ起因の貫通への安価な保険とする(本命は上のポーズ対応。背景は [docs/DEFORMATION_METHODS.md](docs/DEFORMATION_METHODS.md) §5)
- [ ] **マスクペイント** — 修正したくない部位の除外
- [ ] **Before/After 比較** — 修正結果の確認 UI
- [x] **英語版 README** — [README(en).md](README(en).md) を追加し、日本語 README と相互リンク(2026-06-15)。本体ドキュメント(DESIGN 等)は日本語のまま

## 凍結: 差分キャッシュの保存・共有

旧ロードマップにあった「差分キャッシュの保存・再利用」「体型差分プリセットの共有」(旧 FUTURE_PLANS.md の構想を含む)は**凍結**します。

- 体型差分の共有という方向は、変換型の先行ツールが既に担っており機能的な空白がない
- そして保存・配布する差分データは原理的に元アバターの体型情報を含み、No Cache 原則と衝突する(詳細は [docs/DESIGN.md](docs/DESIGN.md) §4)
- 量子化やノイズ付加による「安全化」も検討したが、安全性を保証する形式の議論が固まらない限り実装しない。これが No Cache 原則の由来
